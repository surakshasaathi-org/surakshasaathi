# @suraksha/db

Postgres schema, multi-tenant RLS, and seed data for SurakshaSaathi. Runs on Supabase (India region).

## Layout

```
src/
  schema/
    _shared.ts        enums + helper columns
    tenancy.ts        tenant, app_user, membership
    catalog.ts        insurance_line, product_module, scheme, intake_flow, locale
    user-data.ts      policy, case, case_event, document
    agents.ts         agent_definition, agent_run, review_task
    commerce.ts       feature_flag, entitlement, subscription, payment, affiliate_partner, affiliate_click
    compliance.ts     consent, audit_log, dpdp_request
  seed/               Day-1 seed data (8 modules, 13 insurance lines, 14 agents, 3 schemes)
  client.ts           tenantDb(jwt) for RLS-aware queries; serviceDb() for bypass
  migrate.ts          drizzle-kit migrator
migrations/
  0000_initial.sql    (generated by drizzle-kit generate)
  0001_rls_policies.sql (hand-written — RLS + helpers)
```

## Commands

```bash
pnpm --filter @suraksha/db generate       # regen SQL from schema
pnpm --filter @suraksha/db migrate        # apply migrations
pnpm --filter @suraksha/db seed           # seed Day-1 data (idempotent via onConflictDoNothing)
```

## Adding a new insurance line

One row. That's it.

```ts
await db.insert(insuranceLine).values({
  id: 'crop',
  displayNameI18n: { en: 'Crop Insurance', hi: 'फसल बीमा', kn: 'ಬೆಳೆ ವಿಮೆ' },
  category: 'general',
  enabled: true,
  orderIndex: 120,
});
```
